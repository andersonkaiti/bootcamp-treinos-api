# SQL ou NoSQL?

SQL quando:

- Relacionamento forte entre as entidades.

A aplicação terá: ``Usuário -> Plano de treino -> Dia de treino -> Exercícios``

- É necessário um schema forte, ou seja, tudo segue sempre um padrão.

Um nome sempre será uma ``string``, um peso sempre será um ``number``, etc.

NoSQL quando:

- Sem relacionamento/relacionamento fraco entre as entidades.
- Priorizar leitura em detrimento de escrita.
